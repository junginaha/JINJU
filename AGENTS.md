# 진주.kr 변경 통제 절대 규칙

이 문서는 진주.kr 프로젝트의 최우선 변경 원칙이다. 모든 작업과 배포에 예외 없이 적용한다.

## 1. 요청 범위만 변경

- 사용자가 이번 요청에서 명시한 항목만 수정한다.
- 요청하지 않은 정리, 개선, 리팩터링, 디자인 보정, 문구 수정, 기능 추가, 데이터 수정은 하지 않는다.
- 작업 중 다른 문제를 발견해도 임의로 고치지 않는다.

## 2. 기존 상태 임의 변경 금지

- 댓글, 게시글, 작성자명, 닉네임, 시간, 카테고리, 반응 수치와 운영 데이터를 임의로 추가·삭제·수정하지 않는다.
- `씨앗`, `익명`을 포함한 모든 공개 표기와 문구를 임의로 추가·삭제·변경하지 않는다.
- 인트로, 로고, 색상, 크기, 위치, 애니메이션, 버튼, 레이아웃과 모바일 구성을 임의로 변경하지 않는다.
- 기존 기능, 정책, 필터, 관리자 흐름과 도메인 설정을 임의로 변경하지 않는다.

## 3. 추가 변경은 사전 승인 필수

- 요청을 처리하려면 추가 변경이 꼭 필요한 경우 작업을 멈춘다.
- 변경 이유, 대상 파일, 사용자 화면과 데이터에 미치는 영향을 먼저 설명한다.
- 사용자의 명시적인 허락을 받은 뒤에만 변경한다.
- 편의상 관련 변경을 함께 묶거나 승인 범위를 넓게 해석하지 않는다.

## 4. 배포 원칙

- 배포 전 변경 파일과 실제 차이를 확인해 요청 범위 밖 변경이 0건인지 검수한다.
- 검수와 빌드가 통과한 경우에만 배포한다.
- 운영 배포 대상은 사용자가 지정한 `진주.kr` 메인 사이트와 GitHub `main` 브랜치뿐이다.
- 복구용 ChatGPT Sites 원본은 수정, 업데이트, 재배포, 리다이렉트, 삭제하지 않는다.
- 운영 사이트에 시험용 게시글이나 댓글을 작성하지 않는다.
- 배포 후 요청한 항목만 반영됐는지 확인하고 결과와 남은 일을 보고한다.

## 5. 충돌 시 우선순위

- 이 규칙은 일반적인 개선 제안, 자동 수정, 코드 정리 관행보다 우선한다.
- 사용자의 최신 명시적 지시가 있을 때만 해당 지시 범위 안에서 이 규칙을 갱신한다.
